Introduction à la machine virtuelle

Une machine virtuelle est un environnement informatique isolé qui simule les ressources d’un ordinateur physique, permettant l’exécution d’un système d’exploitation et d’applications. Elle est composée de fichiers de spécification et de configuration et est soutenue par les ressources physiques d’un hôte.

La Virtualisation

La virtualisation est une technologie que vous pouvez utiliser pour créer des représentations virtuelles de serveurs,
de stockage, de réseaux et d'autres machines physiques.
Le logiciel virtuel imite les fonctions du matériel physique pour exécuter plusieurs machines virtuelles sur une seule
machine physique.
Les entreprises ont recours à la virtualisation pour utiliser efficacement leurs ressources matérielles et obtenir
un meilleur rendement de leurs investissements.
Elle alimente également les services de cloud computing qui aident les organisations à gérer leur infrastructure plus efficacement.

émulateur, simulateur et outil de virtualisation

EMULATEUR

Définition :
Un émulateur reproduit fidèlement le comportement et les fonctionnalités d'un système ou d'un matériel
sur une autre plateforme.

Objectif :
Permet d'exécuter des logiciels ou des systèmes conçus pour une architecture différente sur une autre machine.

Caractéristiques :
Reproduit le matériel (processeur, carte graphique, mémoire, etc.) au niveau bas.
Assure une compatibilité quasi complète avec le système émulé.

SIMULATEUR

Définition :
Un simulateur imite le comportement d’un système ou d’un processus dans un environnement contrôlé sans
reproduire nécessairement le matériel sous-jacent.

Objectif :
Étudier et tester des comportements, des modèles ou des algorithmes dans des conditions proches de la réalité.

Caractéristiques :
Ne reproduit pas fidèlement le matériel.
Se concentre sur la modélisation des comportements et des interactions.
Moins précis qu'un émulateur pour reproduire le matériel réel.

OUTIL DE VIRTUALISATION

Définition :
La virtualisation consiste à créer une version logicielle d’un matériel ou d’un système d’exploitation,
appelée machine virtuelle, fonctionnant dans un environnement isolé.

Objectif :
Exécuter plusieurs systèmes d’exploitation ou environnements indépendants sur un même matériel physique.

Caractéristiques :
Crée une machine virtuelle qui fonctionne comme un véritable ordinateur.
Utilise un hyperviseur pour gérer les ressources matérielles (CPU, RAM, etc.) entre plusieurs machines virtuelles.
Plus performant et précis que l’émulation, car il repose sur les ressources matérielles réelles.

Résumé des différences :

Critère Émulateur Simulateur Outil de Virtualisation
Action Reproduit matériel et logiciel fidèlement Reproduit le comportement ou les processus Crée un environnement virtuel isolé
Exemples QEMU, BlueStacks, Dolphin Cisco Packet Tracer, Xcode iOS Simulator VMware, VirtualBox, Docker
Résumé Exécuter un système conçu pour un matériel différent Tester des scénarios ou comportements Créer des environnements indépendants sur un même matériel

Type de virtualisation pour les environnements

Environnement Type de Virtualisation Hyperviseur Cas d'utilisation
VirtualBox Type 2 (hébergée) Logiciel Tests, développement, apprentissage
VMware Workstation Type 2 (hébergée) Logiciel Tests, simulateurs d'environnements multiples
Stratus (ftServer) Type 1 (bare-metal) Bare-metal (direct) Applications critiques, haute disponibilité
VirtualBox et VMware Workstation sont parfaits pour des environnements non critiques, souvent utilisés pour le développement et les tests.
Stratus est conçu pour des environnements critiques où la fiabilité, la haute disponibilité, et la tolérance aux pannes sont indispensables, avec une virtualisation directe sur le matériel.
